# CU Motivation Bot

Telegram бот для мотивации и обучения студентов, предоставляющий структурированный контент по различным модулям и темам.

## 🚀 Возможности

- **Модульная система обучения** - 8 модулей с темами для изучения
- **Интерактивная навигация** - удобные кнопки для перехода между темами
- **Система целей** - пользователи могут ставить и отслеживать цели на семестр
- **Система вопросов и ответов** - возможность задавать вопросы по модулям
- **Мультимедийный контент** - поддержка изображений к постам
- **Отслеживание прогресса** - система отмечает пройденные модули
- **Административные функции** - добавление контента через команды бота

## 🚀 Установка и запуск

### Локальная разработка

1. **Клонируйте репозиторий:**
   ```bash
   git clone <repository-url>
   cd CUMotivationBot
   ```

2. **Установите зависимости:**
   ```bash
   cd tg_bot
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения:**
   Создайте файл `.env` в корне проекта:
   ```env
   TOKEN=your_telegram_bot_token
   DB_HOST=localhost
   DB_PORT=5432
   DB_NAME=motivation_bot
   DB_USER=postgres
   DB_PASSWORD=your_password
   ADMINS=admin_username1,admin_username2
   ```

4. **Настройте базу данных:**
   ```bash
   # Создайте базу данных PostgreSQL
   createdb motivation_bot
   
   # Или используйте Docker для PostgreSQL
   docker run --name postgres -e POSTGRES_DB=motivation_bot -e POSTGRES_PASSWORD=password -p 5432:5432 -d postgres:15
   ```

5. **Подготовьте данные:**
   Убедитесь, что в папке `tg_bot/data/` есть файлы:
   - `posts.csv` - посты с контентом
   - `pictures.csv` - изображения
   - `questions.csv` - вопросы

6. **Запустите бота:**
   ```bash
   cd tg_bot
   python bot.py
   ```

### Docker развертывание

1. **Настройте переменные окружения:**
   Создайте файл `.env`:
   ```env
   TOKEN=your_telegram_bot_token
   PG_DATABASE=motivation_bot
   PG_USER=postgres
   PG_PASSWORD=your_password
   ADMINS=admin_username1,admin_username2
   ```

2. **Запустите с помощью Docker Compose:**
   ```bash
   docker-compose up -d
   ```

## 📊 Структура базы данных

### Таблицы:

- **users** - информация о пользователях и их прогрессе
- **posts** - контент постов по модулям и темам
- **post_images** - изображения к постам
- **module_questions** - вопросы по модулям

## 🤖 Команды бота

### Для пользователей:
- `/start` - начало работы с ботом
- `/data` - просмотр сохраненных данных

### Для администраторов:
- `/add_post модуль|тема|заголовок|содержимое` - добавление поста
- `/add_image модуль|тема|URL_картинки` - добавление изображения

## 📝 Форматы CSV файлов

### posts.csv
```csv
user_id,module,theme,title,content
1,1,1,Заголовок темы,Содержимое темы...
```

### pictures.csv
```csv
module,theme,picture
1,1,https://example.com/image.jpg
```

### questions.csv
```csv
module,question
1,Вопрос по модулю 1?
```

## 🐛 Устранение неполадок

### Проблемы с импортом aiogram:
Если возникает ошибка "Не удается разрешить импорт aiogram.types", убедитесь что:
- Установлена правильная версия aiogram (3.0.0b7)
- Python версии 3.13+
- Все зависимости установлены корректно

### Проблемы с базой данных:
- Проверьте подключение к PostgreSQL
- Убедитесь что переменные окружения настроены правильно
- Проверьте права доступа пользователя БД
