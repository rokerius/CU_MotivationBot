# CU Motivation Bot

Telegram бот для мотивации и обучения студентов, предоставляющий структурированный контент по различным модулям и темам.

## 🚀 Возможности

- **Модульная система обучения** - 8 модулей с темами для изучения
- **Интерактивная навигация** - удобные кнопки для перехода между темами
- **Система целей** - пользователи могут ставить и отслеживать цели на семестр
- **Система вопросов и ответов** - возможность задавать вопросы по модулям
- **Мультимедийный контент** - поддержка изображений к постам
- **Отслеживание прогресса** - система отмечает пройденные модули
- **Административные функции** - добавление контента через команды бота

## 🛠️ Технологии

- **Python 3.13** - основной язык
- **aiogram 3.0.0b7** - Telegram Bot API
- **PostgreSQL** - база данных
- **asyncpg** - асинхронный драйвер PostgreSQL
- **Docker & Docker Compose** - контейнеризация


## 🚀 Установка и запуск

### Docker развертывание

1. **Настройте переменные окружения:**
   Создайте файл `.env`:
   ```env
   TOKEN=your_telegram_bot_token
   PG_DATABASE=motivation_bot
   PG_USER=postgres
   PG_PASSWORD=your_password
   ADMINS=admin_username1,admin_username2
   ```

2. **Запустите с помощью Docker Compose:**
   ```bash
   docker-compose up -d
   ```

## 📊 Структура базы данных

### Таблицы:

- **users** - информация о пользователях и их прогрессе
- **posts** - контент постов по модулям и темам
- **post_images** - изображения к постам
- **module_questions** - вопросы по модулям


## 🤖 Команды бота
#### Старт
- /start
#### Добавить пост (ADMIN)
- /add_post <номер модуля> | <номер темы> | <заголовок> | <содержимое>
#### Добавить к посту картинку, можно несколько (ADMIN)
- /add_image <номер модуля> | <номер темы> | <URL картинки>
#### Добавить вопрос к посту (ADMIN)
- /add_question <номер модуля> | <вопрос>
#### Информация, которую мы храним о текущем пользователе
- /data

```csv
1,Вопрос по модулю 1?
```

## 🔧 Конфигурация

### Переменные окружения:
- `TOKEN` - токен Telegram бота
- `DB_HOST` - хост базы данных
- `DB_PORT` - порт базы данных
- `DB_NAME` - имя базы данных
- `DB_USER` - пользователь базы данных
- `DB_PASSWORD` - пароль базы данных
- `ADMINS` - список администраторов (через запятую)
